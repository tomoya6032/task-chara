# AI支援報告書機能のセットアップ

## 概要
この機能はChatGPT APIを使用して、日報データから月次支援報告書を自動生成します。

## 必要な設定

### 1. OpenAI APIキーの取得
1. [OpenAI Platform](https://platform.openai.com/)でアカウントを作成
2. API Keysセクションで新しいAPIキーを生成
3. APIキーを安全な場所に保存

### 2. 環境変数の設定

#### 開発環境 (.env ファイル)
```bash
OPENAI_API_KEY=your_openai_api_key_here
```

#### 本番環境
環境に応じてOpenAI APIキーを設定してください：

- **Heroku**: `heroku config:set OPENAI_API_KEY=your_key`
- **Docker**: 環境変数として設定
- **その他**: デプロイ環境の設定方法に従う

### 3. Gemの追加（既に追加済み）
```ruby
gem 'ruby-openai'
```

## 機能の使用方法

### 1. 支援報告書の作成
1. サイドバーの「📋 支援報告書」をクリック
2. 「新規作成」ボタンをクリック
3. タイトルと期間（開始日・終了日）を入力
4. 「🤖 AI生成で作成」ボタンをクリック

### 2. AI生成プロセス
- バックグラウンドでChatGPT APIを呼び出し
- 指定期間の日報データを分析
- 体調、気分、活動内容を総合的に評価
- 専門的な支援報告書を自動生成

### 3. 生成ステータス
- **下書き**: 作成したが未生成
- **生成中**: AI処理中（バックグラウンド実行）
- **完成**: 生成完了
- **エラー**: 生成失敗（APIキー不正、ネットワークエラーなど）

## AIプロンプトの特徴

### 分析対象データ
- **活動カテゴリ**: 食事、入浴、外出、作業、休憩など
- **体調・気分**: 1-10のスケール評価
- **疲労度**: 日々の疲労レベル
- **活動内容**: 詳細な活動記録
- **訪問時間**: 支援提供時間

### 生成される報告書内容
- 期間の概要と総評
- 体調・気分の変化分析
- 活動参加状況の評価
- 特記すべき出来事
- 今後の支援方針提案
- 専門的な観察記録

## トラブルシューティング

### エラーステータスになった場合
1. APIキーが正しく設定されているか確認
2. インターネット接続を確認
3. OpenAI APIの利用制限を確認
4. 「AI再生成」ボタンで再試行

## セキュリティ注意事項

1. **APIキーの管理**
   - `.env`ファイルを`.gitignore`に追加
   - 本番環境では環境変数で管理
   - APIキーをコードに直接書き込まない

2. **データプライバシー**
   - OpenAIに送信されるデータは暗号化
   - 個人識別情報は最小限に
   - OpenAIのデータ使用ポリシーを確認

3. **コスト管理**
   - OpenAI APIは使用量に応じて課金
   - Usage limits設定を推奨
   - 定期的に使用量を監視
